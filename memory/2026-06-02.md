# 2026-06-02

## Project setup (with Claude Code)

Got the AutoCAD MCP integration running on Dato's machine.

- Installed prerequisites via winget: **Node.js 24.16 LTS** and **.NET SDK 8** (then **.NET 10**).
- AutoCAD installed here is **2027** (not 2026 as the repo assumed). Also 2018/2021 present.
- AutoCAD 2027 runs on **.NET 10** — its managed assemblies reference `System.Runtime 10.0.0.0`.
  So the plugin had to retarget from `net8.0-windows` → `net10.0-windows`, and the csproj
  AutoCAD DLL HintPaths were repointed from `AutoCAD 2026` → `AutoCAD 2027`.
- `npm install` initially failed (`UNABLE_TO_VERIFY_LEAF_SIGNATURE`) due to a TLS-intercepting
  proxy/AV on this machine. Fixed by running with `NODE_OPTIONS=--use-system-ca`.
- Built the server (`dist/index.js`) and the plugin
  (`autocad-plugin/bin/Release/net10.0-windows/AutoCAD.MCP.Plugin.dll`).
- Created `.env` pointing at AutoCAD 2027 paths.
- Registered the `autocad` MCP server in Claude Code at **user scope** — shows ✓ Connected.
- Set `MCP_AUTOCAD_TOKEN=default-secret-token` as a user env var.

### Auto-load set up (no more NETLOAD needed)
- Created AutoCAD Autoloader bundle at
  `%APPDATA%\Autodesk\ApplicationPlugins\AutoCADMCP.bundle\` with `PackageContents.xml`
  (`LoadOnAutoCADStartup="True"`, `SeriesMin="R26.0"` — AutoCAD **2027** registers as **R26.0**,
  NOT R27; using R27 silently filtered the bundle out) and `Contents\` holding the DLL,
  Newtonsoft.Json.dll, and deps.json. AutoCAD 2027 loads it automatically every startup;
  the plugin's `IExtensionApplication.Initialize()` starts the HTTP server on load.
- Also copied the DLL set to `Desktop\AutoCAD-MCP-Plugin\` for easy manual `NETLOAD` if needed.
- NOTE: when the plugin is rebuilt, re-copy the new DLL into the bundle's `Contents\` folder
  (and Desktop) — the bundle holds copies, not links.
